** Introduction

# In this scenario, one promising approach is to make use of
# simulation to achieve scientific results.

# [HPC, Sustainability and Enviroment]
High-Performance Computing (HPC) infrastructures are critical for
scientific advancement. However, with increasing concerns regarding
environmental sustainability, HPC clusters and data centers are
recognized as major electricity consumers cite:IAE2024. Simulators aim
to address these issues by creating a faithful model of the real
system, researchers can perform predictive analysis and validate
'what-if' scenarios without the operational costs or constraints of
using the actual machine. Using simulations not only lowers the
electricity consumption of HPC systems, but also targets the lack of
reproducibility in HPC cite:repro2024, ensuring a controlled and
reproducible environment for experiments. In this context, SimGrid is
a free and open-source computational framework capable of simulating
several HPC scenarios with reproducibility and versatility, allowing
researchers to simulate large platforms while maintaining accuracy
cite:simgrid2025.


In this work, we present the development of a simulator for the "Parque
Computacional de Alto Desempenho" infrastructure present at INF/UFRGS
cite:LPPD through the SimGrid programmatic interface Simgrid For You
(S4U), focusing on the network topology. We show comparative results
between the actual cluster and the platform created through
simulation, including compute time, network statistics and energy
consumption.

** Platform Creation and Calibration

We selected only the =cei=, =draco=, =tupi= e =poti= homogeneous partitions
from PCAD infrastructure to simulate initially, with the three first
having six nodes and the last having only five. We choose these
partitions as they are the more used ones, aiming to first create a
generic approach that later can be used in the whole cluster. The
methodology for modeling was divided into programmatic platform
generation and system calibration, focusing on hosts computational
power and the network topology. The simulated environment was created
programmatically using the SimGrid for You (S4U) C++ API with SimGrid
v4.1. This approach allowed for the flexible definition of the cluster
hierarchical structure, effectively mirroring the physical
architecture of PCAD.


Several different experiments were executed for the system
calibration. All experiments were run using Guix to ensure
reproducibility. The host calibration for the simulated partitions
consisted on the execution of High-Performance Linpack (HPL)
cite:HPL2003. For each partition this HPL was executed 10 times in two
different machines. The mean results in Gflops was taken as the time
for a node in the partition. For the network we used simple network
tools such as =iperf3= and =ping= to measure information such as
network bandwidth and latency.

The platform validation...


# To ensure the accuracy of the simulator, empirical calibration was
# performed on the real cluster. Host calibration consisted of executing
# the High-Performance Linpack (HPL) benchmark to establish the baseline
# computational speed (FLOP/s) of the individual nodes. Network
# calibration required a two-step approach. First, iperf3 was used to
# measure raw communication throughput, capturing baseline metrics such
# as the ~9413 Mbps mean bandwidth observed on the 10 Gbps
# partitions. Following this, the official SimGrid MPI calibration tools
# were executed to determine internal simulation parameters,
# specifically the bw-factor. This empirical process highlighted the
# inherent complexities of modeling physical networks, as the calculated
# bandwidth factors exhibited significant variance—ranging from 0.23 to
# 3.42—due to real-world network contention, necessitating careful
# parameter selection to validate the model.
